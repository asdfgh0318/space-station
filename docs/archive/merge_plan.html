inmarsat-sniffer

github.com/alphafox02/inmarsat-sniffer
  • Standalone L-band decoder (1525–1559 MHz). STD-C EGC + Aero ACARS / ADS-C / CPDLC simultaneously, single SDR.
  • Up to 27-channel parallel demod, AVX2/SSE/NEON SIMD, --skip-c-channel for Pi.
  • SDR backends: RTL-SDR, HackRF, BladeRF, USRP, Airspy, SDRplay, SoapySDR, VITA-49.
  • Outputs: JSON UDP feed, SBS/BaseStation, MQTT, JAERO format-3, ZMQ audio.
  • Built-in Leaflet map dashboard on port 8888 (--web), optional spectrum tab.
  • add licensed GPL-3.0 — keep as separate binary, do not statically link / vendor source.

2Merge points — where they snap together

Lock-in · roadmap fit

L-band slots into Phase 3

Project plan already lists Phase 3 = L-band (HRPT 1.7 GHz + Hydrogen 21 cm). Inmarsat at 1.5 GHz shares the same RF family — same LNA class, same feed lineage. Adding Inmarsat is a low-risk dry-run for that phase.

tracker roadmap → adopts Inmarsat as Phase 3.0 first-light target
Lock-in · pointing

Geostationary = trivial pointing

From Warsaw (52.23°N, 21.01°E), Alphasat at 25°E sits at fixed AZ ≈ 165°, EL ≈ 27°. The tracker's goto az el already does this — no orbit prediction, no TLE, no scheduling. Just one preset.

controller.py → add target preset · web/app.py → "Park on Alphasat" button
Lock-in · OS prep

Bootstrap already half-ready

scripts/bootstrap_rpi.sh already installs librtlsdr-dev, rtl-sdr tools, and writes the RTL-SDR udev rules / blacklists DVB driver. Need to add: cmake, build-essential, libacars2-dev, libzmq3-dev, libmosquitto-dev.

bootstrap_rpi.sh → +5 packages and a cmake .. && make step
Lock-in · data plane

Sniffer speaks JSON, FastAPI listens

inmarsat-sniffer emits a JSON UDP feed (--feed udp://127.0.0.1:5000) and supports MQTT. FastAPI app already runs a WebSocket loop — add a UDP/MQTT listener task that re-broadcasts decode events to the dashboard.

web/app.py → new /ws/decode channel · tracker/ → no change
Delta · hardware

Antenna feed mismatch

Existing config targets a Ku-band LNB (10 GHz). Inmarsat L-band needs a different feed: modified GPS patch, helix, or L-band patch + LNA. The 60 cm dish is overkill for L-band — a small patch on a tripod will work for first light.

hardware → swap feed · config.yaml → add band profiles (l_band / ku_band)
Delta · UI

Two web servers, one dashboard

Sniffer's own UI is on :8888, FastAPI is on :8080. Cleanest: keep both running, embed sniffer's map as an iframe panel inside FastAPI. Alternative: drop --web and re-render decode events natively. Iframe wins on effort.

web/app.py → add <iframe src=":8888"> tab in the dashboard
Delta · process model

Sidecar, not library

Don't vendor C source into this repo. Add as git submodule at vendor/inmarsat-sniffer/, build with CMake, run as a managed subprocess (or systemd unit). Keeps GPL-3.0 isolated and updates clean via git submodule update --remote.

new file tracker/sniffer.py → spawn / supervise / restart
Risk · CPU

Pi 4 will breathe hard

27-channel parallel demod is heavy. The README explicitly recommends --skip-c-channel for Pi-class hosts (~27% CPU saved). Default to skip-c-channel; expose a toggle in the dashboard for HackRF/full-bandwidth experiments later.

config.yaml → sniffer.skip_c_channel: true default
Risk · license

GPL-3.0 contagion (avoidable)

inmarsat-sniffer is GPL-3.0-or-later. As a separate binary invoked via subprocess + IPC (UDP/MQTT), the FastAPI app stays under whatever license you choose. Do not import its C code, link against its objects, or re-publish modified source bundled inside this repo.

vendor/ via submodule (link, not copy) · root LICENSE unaffected

6Decisions to lock before we start

Q1. Submodule vs. fork? Submodule keeps upstream updates clean, but a fork lets you pin patches. Default: submodule on main until you need to patch.
Q2. Subprocess vs. systemd unit? Subprocess gives you start/stop from the dashboard; systemd is more robust for unattended ops. Default: subprocess first, systemd in Phase D.
Q3. UDP JSON or MQTT for IPC? UDP is zero-config and stdlib-only. MQTT scales but adds mosquitto. Default: UDP JSON.
Q4. Embed sniffer's web UI or rebuild natively? Iframe = same-day feature. Native = months of UI work. Default: iframe.
Q5. Which feed do you actually have / want to buy? GPS patch (cheapest, known-marginal), helix (best, hardest to print), L-band patch + LNA (middle ground). Need to confirm before Phase A bench test.
Q6. Project license? Currently no LICENSE file at root. Pick one — MIT or Apache-2.0 are both compatible with calling a GPL-3.0 sidecar via IPC. Need this nailed down before adding the submodule.
